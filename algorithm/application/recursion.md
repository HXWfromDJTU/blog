# 递归

### 应用
1️⃣ DFS深度优先算法
2️⃣ 前中后序二叉树算法


### 关键点
1️⃣ 找出递推公式
2️⃣ 找到终止条件
3️⃣ 首先必须要找到把大问题分解为多个小问题
4️⃣ 递归递归，调用称之为“递”，返回称之为“归”
5️⃣ 其实，递归的底层思想就是使用了“栈”这个数据结构，递的时候都将结果保存在栈中，然后归的时候再逐个出栈操作

### 注意点
1️⃣ 不要让自己随着代码进入递归思考
2️⃣ 要假设下级子问题已经解决了
3️⃣ 递归代码的书写，要注意栈溢出的问题

### 如何避免陷入死循环
1️⃣ 警惕重复计算：使用变量保存已经递归计算出来的值，避免重复计算
2️⃣ 警惕堆栈溢出：可以声明一个标志位来限制递归的深度
